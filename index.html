agent-ops advisory

I run a real company on a fleet of coding agents. I help your team do the same.

I'm an engineer and founder at a stealth startup. Most of my engineering ships through AI coding agents running in parallel, at production scale, every day.

That only works with an operating model underneath it: isolation so agents don't clobber each other, discipline so their output stays mergeable, and guardrails so nobody has to babysit. I advise engineering teams on building that model. I sell judgment, not a product.

scope

What I help with

Concrete, operational, and earned in production. Not slideware about "AI strategy."

isolation

Parallel worktree isolation

Give every agent its own checkout, ports, and database slice so a dozen of them can work at once without colliding on state or stepping on each other's branches.

merge discipline

Keeping stacked PRs alive

A stack of dependent PRs that survives reviews, rebases, and merges below it without dissolving into conflict resolution. The discipline that makes agent throughput actually land.

dispatch

Dispatching and supervising workers

Hand work to agent workers, recover cleanly when one crashes mid-task, and supervise the fleet without becoming the bottleneck. Crash-recoverable by design, not by luck.

guardrails

CI guardrails that keep output honest

The checks, gates, and migration discipline that catch agent mistakes before a human does, so velocity never comes at the cost of a trustworthy main branch.

observability

Single-screen fleet observability

One view of what every agent is doing, where it's stuck, and what it shipped, so running ten workers feels like running one.

operating model

The model that ties it together

How these pieces compose into a repeatable way of working that your existing engineers can run, adopt incrementally, and own after I'm gone.
